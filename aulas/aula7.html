Aula: Diagrama de Sequência (UML)

1. O que é um Diagrama de Sequência?

O diagrama de sequência mostra como os objetos interagem ao longo do tempo.

Diferente do diagrama de classes (estrutura), aqui você vê:

  • quem conversa com quem
  • em que ordem
  • o que é enviado (mensagens)
Pense como um roteiro de filme:
“Usuário clica → sistema valida → banco responde → sistema retorna”

2. Elementos principais

Ator

Quem inicia a interação (ex: Usuário)

Objetos / Participantes

  • Sistema
  • Controller
  • Banco de Dados

Linha de vida

Linha vertical que representa o tempo daquele objeto

Mensagens

  • Síncrona (espera resposta)
  • Assíncrona (não espera)
  • Retorno (linha tracejada)

Ativação

Mostra quando o objeto está executando algo

3. Exemplo: Login

Cenário

Usuário faz login no sistema

Diagrama em PlantUML

@startuml
actor Usuario

participant "Tela Login" as Tela
participant "Sistema" as Sistema
participant "Banco de Dados" as BD

Usuario -> Tela : Digita login/senha
Tela -> Sistema : enviarCredenciais()
Sistema -> BD : validarUsuario()

BD --> Sistema : resultado

alt Login válido
    Sistema --> Tela : sucesso
    Tela --> Usuario : acesso liberado
else Login inválido
    Sistema --> Tela : erro
    Tela --> Usuario : mensagem de erro
end

@enduml

4. Explicação passo a passo

  1. Usuário envia dados
  2. Tela repassa ao sistema
  3. Sistema consulta banco
  4. Banco responde
  5. Sistema decide (válido ou não)
  6. Retorna ao usuário

5. Estruturas importantes

ALT (if/else)

alt condição
else outra condição
end

LOOP

loop enquanto condição
end

OPT (opcional)

opt condição
end

6. Quando usar?

  • Detalhar um caso de uso
  • Entender fluxo entre camadas
  • Explicar integração entre sistemas
  • Modelar APIs

Lista de Exercícios – Diagramas de Sequência (UML)

Exercício 1 – Login simples

Cenário: Um usuário acessa um sistema e realiza login informando usuário e senha.

Requisitos:

  • Usuário envia login e senha
  • Sistema valida as credenciais
  • Sistema retorna sucesso

Faça o diagrama apenas para o caso de sucesso.

Exercício 2 – Login com erro

Cenário: Login com possibilidade de erro de autenticação.

Requisitos:

  • Usuário informa login e senha
  • Sistema valida os dados
  • Em caso de erro, exibir mensagem
  • Usuário permanece na tela de login

Utilize o fragmento alt.

Exercício 3 – Cadastro de usuário

Cenário: Um usuário realiza seu cadastro no sistema.

Requisitos:

  • Usuário informa os dados
  • Sistema valida as informações
  • Um novo usuário é criado
  • Os dados são armazenados no banco

Exercício 4 – Consulta de produto

Cenário: Cliente consulta um produto pelo código.

Requisitos:

  • Cliente informa o código do produto
  • Sistema busca no banco de dados
  • Produto é retornado ou mensagem de erro é exibida

Utilize o fragmento alt.

Exercício 5 – Compra de produto

Cenário: Cliente realiza a compra de um produto disponível em estoque.

Requisitos:

  • Cliente seleciona o produto
  • Sistema verifica o estoque
  • Pedido é criado
  • Estoque é atualizado

Exercício 6 – Compra com estoque insuficiente

Cenário: Compra de produto com possibilidade de falta de estoque.

Requisitos:

  • Sistema verifica o estoque
  • Se não houver quantidade suficiente, a compra é cancelada
  • Cliente é informado

Utilize o fragmento alt.

Exercício 7 – Pagamento com cartão

Cenário: Cliente realiza pagamento com cartão de crédito.

Requisitos:

  • Sistema envia dados ao gateway de pagamento
  • Pagamento é autorizado ou negado
  • Sistema registra o resultado

Exercício 8 – Recuperação de senha

Cenário: Usuário solicita recuperação de senha.

Requisitos:

  • Usuário informa o e-mail
  • Sistema valida o e-mail
  • Sistema gera um token
  • E-mail de recuperação é enviado

Exercício 9 – Pedido com múltiplos itens

Cenário: Cliente monta um pedido com vários produtos.

Requisitos:

  • Cliente adiciona itens ao pedido
  • Para cada item, o estoque é verificado
  • Item é adicionado ao pedido

Utilize o fragmento loop.

Exercício 10 – Caso completo (desafio)

Cenário: Processo completo de compra em um e-commerce.

Inclua no diagrama:

  • Login do usuário
  • Consulta de produtos
  • Adição ao carrinho
  • Pagamento
  • Confirmação do pedido

Utilize fragmentos alt, loop e criação de objetos.
