Aula 02 – Introdução ao PlantUML no Visual Studio Code

  • Entender o que é PlantUML e por que usar em Engenharia de Software.
  • Instalar e configurar PlantUML no VS Code (extensão + Java + Graphviz).
  • Criar diagramas: Caso de Uso, Sequência e Atividade.
  • Exportar imagens e integrar com o repositório GitHub.

1. O que é PlantUML?

PlantUML é uma linguagem textual para criar diagramas UML. Em vez de desenhar manualmente, você descreve o diagrama em texto e o PlantUML gera a imagem. É perfeito para versionar em Git, revisar por PR e manter documentação enxuta.

  • Suporta: Casos de Uso, Atividade, Sequência, Classe, Componentes, Estados, entre outros.
  • Vantagens: rastreável, colaborativo, automatizável e simples de manter.

2. Instalação e Configuração

2.1 Extensão do VS Code

  • Abrir o VS Code → Extensions (Ctrl+Shift+X).
  • Buscar por PlantUML e instalar a extensão (autor: jebbs).

2.2 Java (obrigatório)

Instale o OpenJDK 17 (ou compatível). Verifique no terminal:

java -version

2.3 Graphviz

Instale o Graphviz para renderização de certos diagramas. Após instalar, verifique:

dot -V

3. Seu primeiro diagrama

Crie um arquivo diagrama.puml e cole:

@startuml
Alice -> Bob: Olá Bob
Bob --> Alice: Oi Alice!
@enduml

Visualize com Alt + D (Preview) ou pelo botão Preview Current Diagram.

4. Exemplos prontos

4.1 Caso de Uso

@startuml
actor "Usuário" as U
rectangle Sistema {
  U --> (Login)
  U --> (Realizar Pedido)
  U --> (Consultar Histórico)
}
@enduml

4.2 Sequência

@startuml
actor Cliente
participant Sistema
participant "Serviço de Pagamento" as Pag

Cliente -> Sistema: Inicia Compra
Sistema -> Pag: Solicita Pagamento
Pag --> Sistema: Confirmação
Sistema --> Cliente: Compra Aprovada
@enduml

4.3 Atividade

@startuml
start
:Cliente acessa o app;
:Seleciona produto;
:Confirma compra;
if (Pagamento aprovado?) then (sim)
  :Exibe recibo;
else (não)
  :Exibe erro;
endif
stop
@enduml

5. Organização no repositório

/design
  /uml
    caso-uso.puml
    atividade-login.puml
    sequencia-compra.puml

Para incluir imagens em docs, exporte o diagrama (botão direito → Export Current Diagram) em .png ou .svg.

6. Integração com GitHub

  • Faça commit dos arquivos .puml e, se necessário, das imagens exportadas.
  • Abra PR para revisão do conteúdo (texto do diagrama é revisável!).
  • Referencie imagens em Markdown (README, Wiki):
![Sequência de Compra](design/uml/sequencia-compra.png)
