1) O que é um Caso de Uso?

Um Caso de Uso descreve como um ator (usuário ou sistema externo) interage com o sistema para atingir um objetivo específico. Ele narra um fluxo completo, do início ao fim.

3) Exemplo completo — Caso de Uso: Realizar Login

Caso de Uso: Realizar Login
Ator: Usuário
Objetivo: Acessar o sistema mediante autenticação válida

Pré-condições:
- Possuir conta ativa

Pós-condições:
- Sessão iniciada

Fluxo Principal:
1) Usuário informa e-mail e senha.
2) Sistema valida credenciais.
3) Sistema redireciona para a tela inicial.

Fluxos Alternativos:
A1 - Credenciais inválidas:
  1) Sistema rejeita credenciais.
  2) Exibe mensagem de falha.
  3) Volta ao passo 1 do fluxo principal.

A2 - Usuário bloqueado:
  1) Sistema detecta tentativas excedidas.
  2) Informa bloqueio temporário.

Regras de Negócio:
- RN01: Bloqueio após 5 tentativas inválidas.

Requisitos Relacionados:
- RF01 Login
- RNF01 Resposta em até 2s

5) Como um Caso de Uso vira User Stories e MVP

Exemplo: Caso de Uso “Realizar Pedido”

Este caso de uso é perfeito para demonstrar como funcionalidade → stories → MVP se conectam.

User Stories Essenciais (MVP)

US01 — Selecionar produto
Como cliente, quero visualizar os produtos para escolher o que comprar.

US02 — Adicionar ao carrinho
Como cliente, quero adicionar itens ao carrinho.

US03 — Fechar pedido
Como cliente, quero revisar meu carrinho antes de confirmar.

US04 — Escolher forma de pagamento
Como cliente, quero selecionar uma forma de pagamento.

US05 — Confirmar pedido
Como cliente, quero confirmar o pedido e receber a confirmação.

User Stories Complementares (Pós-MVP)

US06 — Calcular frete  
US07 — Aplicar cupom de desconto  
US08 — Acessar histórico de pedidos  
US09 — Rastrear pedido  
US10 — Avaliar produtos

MVP na prática

O MVP contém apenas as stories essenciais que executam todo o fluxo principal do caso de uso:

MVP:
- US01 Selecionar produto
- US02 Adicionar ao carrinho
- US03 Fechar pedido
- US04 Escolher pagamento
- US05 Confirmar pedido
