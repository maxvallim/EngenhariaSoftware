Diagramas de Classe e Conceitos de OO

Propósito

Fornecer um guia rápido com os comandos essenciais do PlantUML para criar Diagramas de Classe, aplicando conceitos de Orientação a Objetos como encapsulamento, herança, polimorfismo e abstração.

1) Estrutura básica

@startuml
... elementos ...
@enduml

2) Definição de Classe

class Usuario
class Produto
class Pedido

3) Atributos e Métodos

class Usuario {
  -id: int          ' privado
  -nome: string     ' privado
  +login(): boolean ' público
  +logout(): void   ' público
}

4) Encapsulamento

class ContaBancaria {
  -saldo: float
  +depositar(valor: float): void
  +sacar(valor: float): boolean
}

Encapsulamento protege os atributos internos e expõe apenas métodos controlados.

5) Herança

abstract class Pessoa {
  +nome: string
  +falar(): void
}

class Cliente
class Funcionario

Cliente --|> Pessoa
Funcionario --|> Pessoa

6) Interfaces e Polimorfismo

interface Autenticavel {
  +autenticar(): boolean
}

class Usuario
Usuario ..|> Autenticavel

Interfaces permitem polimorfismo: diferentes classes podem implementar o mesmo contrato.

7) Multiplicidade e Associações

Usuario "1" --> "0..*" Pedido
Pedido "1" --> "1..*" Produto

8) Composição e Agregação

class Motor
class Carro

Carro *-- Motor      ' composição (Motor não existe sem Carro)
Carro o-- Usuario    ' agregação (Usuário pode existir sem Carro)

9) Enumerações

enum StatusPedido {
  PENDENTE
  PAGO
  CANCELADO
}

10) Exemplo completo

@startuml
abstract class Pessoa {
  +nome: string
  +falar(): void
}

class Cliente {
  -id: int
  +realizarPedido(): Pedido
}

class Funcionario {
  -matricula: int
  +processarPedido(p: Pedido): void
}

class Pedido {
  -numero: int
  -data: Date
  +finalizar(): void
}

class Produto {
  -codigo: int
  -descricao: string
  +calcularPreco(): float
}

enum StatusPedido {
  PENDENTE
  PAGO
  CANCELADO
}

Pessoa <|-- Cliente
Pessoa <|-- Funcionario
Cliente "1" --> "0..*" Pedido
Pedido "1" --> "1..*" Produto
Pedido --> StatusPedido
@enduml
