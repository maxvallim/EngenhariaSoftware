Aula 05 — Diagramas de Atividade (UML)

Objetivos da aula

  • Entender para que servem Diagramas de Atividade.
  • Relacionar Caso de Uso → Fluxo Narrativo → Atividade.
  • Aprender os principais elementos: início, ações, decisões, forks, joins e swimlanes.
  • Criar Diagramas de Atividade utilizando PlantUML.
  • Gerar arquivos .puml organizados no repositório GitHub.

1) O que é um Diagrama de Atividade?

O Diagrama de Atividade representa o fluxo de ações, decisões e caminhos alternativos dentro de um processo. Ele é o diagrama mais adequado para visualizar a lógica do Caso de Uso.

É excelente para:

  • Transformar fluxos narrativos em algo visual;
  • Identificar gargalos e complexidades;
  • Preparar diagramas de sequência;
  • Modelagem de negócios, UX e processos.

2) Principais elementos

  • Nó inicial
  • Ação
  • Decisão (if/else)
  • Fork / Join
  • Swimlanes (faixas de responsabilidade)
  • Nó final

3) Exemplo — Fluxo simples (Login)

@startuml
start
:Usuário informa e-mail e senha;
:Sistema valida credenciais;

if (Credenciais válidas?) then (sim)
  :Redireciona para dashboard;
else (não)
  :Exibe mensagem de erro;
endif

stop
@enduml

4) Exemplos com Swimlanes

Swimlanes separam quem executa cada ação. Excelente para mostrar responsabilidades.

4.1) Login com Swimlanes

@startuml
|Usuário|
start
:Informa e-mail e senha;

|Sistema|
:Valida credenciais;

if (Válidas?) then (sim)
  |Sistema|
  :Cria sessão;
  |Usuário|
  :Acessa dashboard;
else (não)
  |Sistema|
  :Exibe erro de login;
endif

stop
@enduml

4.2) Fluxo de Pedido (Cliente x Sistema x Pagamento)

@startuml
|Cliente|
start
:Seleciona produtos;
:Confirma carrinho;

|Sistema|
:Calcula frete;
:Envia dados do pagamento;

|Gateway de Pagamento|
:Processa pagamento;

if (Aprovado?) then (sim)
  |Sistema|
  :Gera pedido;
  |Cliente|
  :Exibe confirmação;
else
  |Cliente|
  :Exibe falha no pagamento;
endif

stop
@enduml

5) Ações paralelas (fork/join)

@startuml
start
:Iniciar fluxo;

fork
  :Enviar e-mail ao cliente;
fork again
  :Atualizar controle de estoque;
fork again
  :Gerar log da operação;
end fork

:Finalizar processo;
stop
@enduml
