Cheatsheet — PlantUML

Diagramas de Sequência

Propósito

Fornecer um guia rápido com os comandos essenciais do PlantUML para criar Diagramas de Sequência, mostrando a ordem de interação entre atores, objetos e camadas do sistema.

1) Estrutura básica

@startuml
... elementos ...
@enduml

2) Participantes

actor Usuario
participant Sistema
participant "Banco de Dados" as BD
participant Controller
participant Service

3) Mensagens

Usuario -> Sistema : solicitarLogin()
Sistema --> Usuario : resposta

4) Fluxo básico de interação

@startuml
actor Usuario
participant Sistema
participant BD

Usuario -> Sistema : informarCredenciais()
Sistema -> BD : validarUsuario()
BD --> Sistema : ok
Sistema --> Usuario : acessoLiberado()
@enduml

5) Ativação (foco de execução)

Usuario -> Sistema : login()
activate Sistema
Sistema -> BD : consultar()
activate BD
BD --> Sistema : dados
deactivate BD
Sistema --> Usuario : resposta
deactivate Sistema

6) Condição (alt / else)

alt Credenciais válidas
  Sistema --> Usuario : acesso permitido
else Credenciais inválidas
  Sistema --> Usuario : erro de login
end

7) Fluxo opcional (opt)

opt Usuário esqueceu a senha
  Usuario -> Sistema : solicitarRecuperacao()
  Sistema --> Usuario : email enviado
end

8) Repetição (loop)

loop enquanto houver itens
  Sistema -> BD : processarItem()
end

9) Mensagens assíncronas

Sistema ->> Fila : publicarEvento()

10) Criação e destruição de objetos

Sistema -> Pedido : <<create>>
destroy Pedido

11) Notas

note right of Sistema
 Validação ocorre
 com base no CPF
end note

Sistema .. note right : Camada de serviço

12) Organização visual

left to right direction

box "Camada de Serviço"
  participant Controller
  participant Service
end box

13) Exemplo completo e funcional

@startuml
actor Cliente
participant "Web App" as App
participant Controller
participant Service
participant "Banco de Dados" as BD

Cliente -> App : clicarComprar()
App -> Controller : enviarPedido()

activate Controller
Controller -> Service : processarPedido()
activate Service

Service -> BD : salvarPedido()
BD --> Service : ok

Service --> Controller : pedidoCriado
deactivate Service

Controller --> App : confirmação
deactivate Controller

App --> Cliente : pedido realizado
@enduml
